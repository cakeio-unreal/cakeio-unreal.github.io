Outcomes

Overview

Source Location: Outcomes

The following types are all defined in the following header: 

#include "CakeIO/CakeOutcomes.h"

Outcomes are special enums that represent the final outcome of a desired IO operation. They are used to give enhanced context about the final outcome of an IO operation.

IO Outcomes

There are two outcome types for IO operations: one for file operations and one for directory operations. Before we examine each outcome type, we must first cover two common values that these outcome types share.

Ok and No Op

Both file and directory outcome types have two values that do not indicate an actual error: Ok and No Op. It is important to understand the difference between these two outcomes.

Ok: The IO operation was executed and encountered no errors.

No Op: The IO operation was skipped because it would have resulted in no effect.

The primary reason NoOp exists is so that callers can know whether or not an IO operation actually occurred. It is up to the caller to determine whether or not distinguishing between Ok and NoOp is important. For example, if we want to delete a file, we might decide that we don't care whether or not the delete operation actually occurred, and all we actually care about is that the file does not exist on the filesystem. In this case, we can consider Ok and NoOp as identical. As another example, let's say that we are allowing users to append text data to a file via a GUI. In this situation, we might decide to check for NoOp, which is generated whenever an append operation is sent empty data. Not all IO operations will generate a NoOp. Refer to the Error Maps to know the possible outcomes each IO operation can generate.

ECakeOutcomeFileIO

This is the outcome value that is dedicated to represent outcomes associated with IO operations involving an individual file.

Enum Value Display Name Description
Ok Ok The operation was successful.
NoOp No Op The operation was skipped because it would have accomplished nothing.
BadFileName Bad File Name The file name is invalid.
DoesNotExist Does Not Exist The file does not exist.
DestDirDoesNotExist Dest Dir Does Not Exist The destination directory for the file does not exist.
AlreadyExists Already Exists The file already exists.
FailedStatQuery Failed Stat Query Failed to retrieve OS stat data for the file.
FailedOpenRead Failed Open Read The file could not be opened in read-only mode.
FailedOpenWrite Failed Open Write The file could not be opened in write-only mode.
FailedOpenReadWrite Failed Open Read Write The file could not be opened in read-write mode.
IsReadOnly Is Read Only The file is read-only and cannot be modified.
CouldNotChangePerms Could Not Change Perms Could not change permissions for the file.
FailedRead Failed Read Failed to read the file's data.
FailedWrite Failed Write Failed to write data to the file.
FailedDelete Failed Delete Failed to delete the file.

ECakeOutcomeDirIO

This is the outcome value that is dedicated to represent outcomes associated with IO operations involving an individual directory.

Enum Value Display Name Description
Ok Ok The operation was successful.
NoOp No Op The operation was skipped because it would have accomplished nothing.
BadDirName Bad Dir Name The directory name is invalid.
DoesNotExist Does Not Exist The directory does not exist.
DestDirDoesNotExist Dest Dir Does Not Exist The destination directory does not exist.
AlreadyExists Already Exists The directory already exists.
FailedStatQuery Failed Stat Query Failed to retrieve OS stat data for the directory.
FailedCreate Failed Create The directory could not be created.
FailedDelete Failed Delete Failed to delete the directory.
FailedCopy Failed Copy Failed to copy the directory.

File / Directory Name Sanitization

Both file and directory outcomes have a value that represents an invalid file or directory name. Right now this value is returned only in functions that involve changing the file or directory name, and it is returned when the new file/directory name is empty or contains only whitespace and/or path separators. There is no platform specific validation that occurs regarding illegal characters in paths.

FailedDeletingPreexisting

While most outcomes are self-explanatory, this particular outcome value could benefit from a little extra explanation. This outcome can occur when the user runs an IO operation that allows preexisting items to be overwritten by the IO operation. In the event that an item already exists at the destination path, the IO operation will attempt to first delete that item in order to ensure the main IO operation can proceed. If the item at the destination path cannot be deleted, this value is returned.

Directory Traversal Outcomes

Note

This section deals with directory traversal. If you are unfamiliar with this concept, please see this section.

The following outcome enums are used to represent the result of directory traversal.

Did Not Launch

All traversal outcomes have a value to indicate that a traversal operation failed to start. It is important to understand the contexts that can cause a traversal to fail to start:

  1. The source directory does not exist. When a traversal is called on a CakeDir object, that traversal operation will first ensure that the directory it represents actually exists on the filesystem. If it does not, the traversal will fail to launch.

  2. A filtered traversal is called on a CakeDir object when its extension filter is empty. This would visit files in a way that the caller does not want, and so the traversal will fail to launch.

  3. A policy argument with an out of range value was submitted. All Cake IO policies are enums, so out of range issues should be quite rare. The only way these are able to occur is by casting integer values to the policy type. When an out of range policy argument is encountered, the traversal will fail to launch.

Tip

If Cake IO logging is enabled, it will describe the error that prevented the traversal from launching.

ECakeOutcomeTraversal

This outcome value is used by both unguarded and guarded traversal styles.

Enum Value Display Name Description
DidNotLaunch Did Not Launch The traversal was unable to begin.
Completed Completed The traversal completed without incident.
Aborted Aborted The traversal was aborted due to an error.

ECakeOutcomeSearch

This outcome value is used only by search traversals.

Enum Value Display Name Description
DidNotLaunch Did Not Launch The traversal was unable to begin.
Succeeded Succeeded The search satisfied its requirements.
Failed Failed All elements were visited and the search failed to satisfy its requirements.
Aborted Aborted The traversal was aborted due to an error.

Advanced Outcomes

ECakeOutcomeDirWork

This outcome value that is used by higher level functions that perform some work on elements within a source directory. Used extensively in CakeMix functions.

Enum Value Display Name Description
WorkCompleted Work Completed The directory work completed successfully.
SourceDirDoesNotExist Source Dir Does Not Exist The submitted source directory does not exist and the directory work was unable to begin.
TraversalDidNotLaunch Traversal Did Not Launch The traversal operation was unable to launch.
Aborted Aborted The directory work was aborted due to an error.

ECakeOutcomeBatchOp

This outcome value that is used by CakeAsyncIO's batch operations.

Enum Value Display Name Description
Ok Ok The batch operation completed successfully.
NoOp No Op The batch operation was submitted an empty batch and was skipped.
Aborted Aborted The batch operation was aborted due to an error.
